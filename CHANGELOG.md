# Changelog

This changelog summarizes release-level milestones. Git history, machine-readable contracts,
append-only research memory, and signed proof bundles remain the authoritative record of exact
changes and evidence.

### Admission authority engineering

- Added implementation-only Shortcut Authority V2 tree, cross-fit, provenance, and encrypted target

### Credibility controls

- Bound the current claim registry to exact reviewed claim semantics, explicit lineage, normalized
  repository evidence paths, and committed evidence content.
- Made generated-handoff rendering independent of ambient import order and rejected unreviewed
  preloaded package modules.
- Moved recovery rendering into a private no-hardlink committed snapshot and a fresh, bounded child
  process with parent-side source and input rechecks.
- Bound every materialized recovery file to the selected commit's exact path, content, size, and
  executable mode, rejecting ignored, untracked, dirty, linked, and special-file inputs.
- Added a canonical same-operator engineering-validation receipt whose lineage, command plan, raw
  artifacts, JUnit counts, coverage counts, and mission inventory are recomputed during recovery.
- Required xfail-marked cases to execute normally, zero skipped receipt tests, strict UTF-8 JUnit
  evidence, and an exact 90.01 percent statement-plus-branch coverage floor.
- Required the final recovery ledger to be a strict byte-prefix append of its evidence parent and
  bound both final recovery files to regular nonexecutable Git blobs.
- Aligned the public research state, acquisition thresholds, planner units, and product stages with
  executable boundaries. Added a prospective exposure-normalized value refinement that still
  requires a value-stage preregistration or amendment and does not modify the frozen master formula.

No scientific performance, safety, transfer, product-readiness, or economic claim has been made.
