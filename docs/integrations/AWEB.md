# Aweb Integration Gate

Aweb and Maestro are optional future adapters, not dependencies or ownership boundaries.

No adapter will be added until runtime discovery returns a callable research execution capability
with persistence, independent verification, artifact collection, approval enforcement, and an
end-to-end receipt test. Capability slugs and provider IDs must be discovered at runtime; they are
never invented or hard-coded from historical documents.

Until that gate passes, this repository emits provider-neutral job specifications and runs the
scientific path locally. Aweb may assist with governed discovery or metadata inspection without
becoming a source of scientific truth.
